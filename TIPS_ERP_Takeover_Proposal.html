Executive Summary

The Indian Public School (TIPS) Group currently operates a Microsoft-based ERP solution managed by an existing vendor. TIPS now seeks to transition full ownership of the ERP system — including source code, customisations, configurations, and technical documentation — to a new trusted technology partner.

N2P Systems proposes a structured ERP Analysis and Code Takeover engagement to achieve this transition safely, thoroughly, and with zero disruption to daily school operations.

Our commitment: TIPS will emerge from this engagement with complete ownership, full documentation, and a prioritised improvement roadmap for their Microsoft Dynamics ERP — with N2P Systems as the accountable technical partner.

Context & Objectives

TIPS Group has invested significantly in a Microsoft-based ERP tailored for educational administration — covering fee management, HR & payroll, procurement, academic scheduling, and financial reporting across multiple campuses. The need to transition ERP ownership can arise from vendor relationship changes, lack of documentation, unresolved technical debt, or the desire for greater institutional control.

2.1 Engagement Objectives
  • Conduct a comprehensive technical analysis of the existing ERP — architecture, codebase, customisations, integrations, and data model
  • Identify technical gaps, risks, undocumented logic, and areas requiring remediation
  • Formally acquire full ownership of source code, configurations, and deployment artefacts
  • Establish complete technical documentation and a knowledge base for the TIPS ERP environment
  • Transition all support and development responsibilities to N2P Systems with zero operational disruption
  • Provide TIPS management with full visibility and control over their ERP asset
2.2 Key Challenges We Address
Undocumented CustomisationsMapping all bespoke code and configurations built on top of the Microsoft base product
Knowledge GapCapturing logic held only by the outgoing vendor or individual developers
Integration ComplexityUnderstanding all third-party connections (payment gateways, attendance, HRMS, etc.)
Data IntegrityValidating that production data is consistent, backed up, and recoverable
Licensing & AccessEnsuring N2P obtains all necessary source access, credentials, and license artefacts
Business ContinuityCompleting the transition without disrupting academic calendars, fee cycles, or payroll runs

Scope of Work

The engagement is structured into four sequential phases. Each phase has defined entry and exit criteria — no phase is concluded without verifiable completeness.

1
Weeks 1–3
Discovery & System Analysis
3 Weeks · Fixed
  • Stakeholder kick-off workshops (IT, Finance, HR, Ops)
  • Architecture & hosting environment review
  • Full module inventory across all campuses
  • Third-party integration mapping
  • Existing documentation review
  • IP ownership & vendor contract assessment
2
Weeks 4–6
Code Audit & Gap Assessment
3 Weeks · Fixed
  • Full custom codebase walkthrough & review
  • Undocumented business rules identification
  • Code quality & maintainability assessment
  • Security audit (SQL injection, exposed creds)
  • Performance audit (slow queries, batch jobs)
  • Formal Code Audit Report with severity ratings
3
Weeks 7–10
Code Takeover & Documentation
4 Weeks · Fixed
  • Source code transfer to secure N2P repository (Azure DevOps)
  • Full system architecture documentation
  • Deployment runbooks & rollback procedures
  • Credential & API key secure handover
  • Change management & branching strategy setup
  • Database backup validation & DR baseline
4
Weeks 11–14
Stabilisation & Handover
4 Weeks · Fixed
  • Shadow support alongside outgoing vendor
  • Parallel running & incident validation
  • Formal cut-over — N2P assumes primary ownership
  • SLA obligations commence
  • Post-takeover health review
  • Prioritised remediation backlog to management
Out of scope: Full redevelopment of ERP modules, new feature development, end-user retraining, hardware procurement, and Microsoft licensing are excluded from this engagement and will be quoted separately if required.

Risk Register & Mitigation

N2P Systems has identified the following key risks and the corresponding mitigations built into our engagement methodology.

Risk Severity Mitigation Strategy
Incomplete source code handover by outgoing vendor High Legal review of IP clauses in existing contracts; N2P can independently reconstruct missing components from running environment
Undocumented business logic or critical workarounds High Structured interviews with power users and TIPS staff; testing against known business scenarios to validate all captured logic
Data integrity issues in production database High Full database backup and validation before any takeover activity; restore tests performed in a dedicated staging environment
Disruption to live operations during transition Medium All takeover activities performed in non-production environments first; production changes scheduled during off-peak windows only
Security vulnerabilities in inherited codebase Medium Dedicated security audit in Phase 2; all critical vulnerabilities remediated before formal go-live as primary owner
Delayed cooperation from outgoing vendor Medium Parallel access strategy — N2P can independently analyse running environments without full outgoing vendor cooperation
Knowledge loss from key TIPS internal staff Low Knowledge base built iteratively during Phase 1; all institutional knowledge captured in documentation before Phase 3

Assumptions, Exclusions & Terms
9.1 Assumptions
  • TIPS Group holds or will obtain IP ownership rights to the existing ERP codebase prior to or during this engagement
  • TIPS will ensure N2P Systems receives read access to all production and non-production environments required for analysis
  • The outgoing vendor will be available for a minimum of 4 structured handover sessions during Phase 3
  • TIPS IT coordinator and functional leads from Finance, HR, and Operations will participate in Phase 1 workshops
  • Existing Microsoft Dynamics licenses and Azure subscriptions will be maintained by TIPS throughout and after the takeover
9.2 Exclusions
  • Renegotiation with the outgoing vendor or legal proceedings to obtain source code (legal support billed separately if required)
  • Full redevelopment or re-implementation of ERP modules (new SOW required post-audit if needed)
  • New module development or major functional enhancements (billable via CR process post-takeover)
  • Hardware or infrastructure procurement
  • End-user retraining programs
9.3 Contract Terms
  • Fixed-scope, fixed-price engagement — changes agreed via formal Change Request only
  • Either party may suspend with 14 days written notice in the event of a material breach
  • All client data and business logic are treated as strictly confidential throughout and after the engagement
  • Source code and documentation produced in Phase 3 are held with TIPS as the primary IP owner
Proposal validity: This proposal is valid for 30 days from March 15, 2026 (valid until April 14, 2026). Pricing and timelines are subject to revision based on system complexity discovered during Phase 1, if materially different from expectations.
